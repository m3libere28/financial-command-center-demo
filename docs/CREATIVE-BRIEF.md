# Public Demo Creative Brief

## Purpose

Help a first-time visitor understand how a connected financial command center can clarify cash flow, investments, goals, and a major life transition.

## Story

The Morgan household is entirely fictional. Its financial data exists only to demonstrate product relationships and decision support. Runtime values in `data.js` are authoritative.

## Experience principles

- Calm command: premium clarity, restraint, and visible hierarchy.
- Financial meaning before decoration.
- Trust before novelty.
- Motion should communicate state and respect reduced-motion preferences.
- Mobile and keyboard operation are first-class requirements.
- Copilot responses remain scripted, fictional, and educational.

## Safety requirements

- Keep fictional disclosure visible.
- Do not use personal-repository data, assets, labels, or screenshots.
- Do not add production integrations, visitor-data collection, tracking, or persistent storage.
- Deploy only the explicit runtime allowlist from `pages.yml`.
